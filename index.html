Istoric

Inainte de epoca moderna, criptografia se ocupa doar cu asigurarea confidentialitatii mesajelor (criptare) — conversia de mesaje dintr-o forma clara într-una neclara, si inversul acestui proces, pentru a face mesajul imposibil de înteles pentru cei care intercepteaza mesajul si nu au cunostinte secrete aditionale (anume cheia necesara pentru decriptarea mesajului). In ultimele decenii, domeniul s-a extins dincolo de problemele de confidentialitate si include, printre altele, si tehnici de verificare a integritătii mesajului, autentificare a trimitatorului si receptorului, semnatură electronica, calcule securizate.

Vigenère Cipher

O cripare monoalfabetica primeste ca input doua stringuri, unu find plaintextul care trebuie encriptat al doilea find chaia de encripate. Daca cheia de encripare este mai mica decat plaintextul se expandeaza concatenand-o pana ajunge la dimensiunea plaintextului ori daca este mai mare trunchind-o. Dupa aceasta se efectueaza pentru fiecare litera in parte din plaintext si cheie dupa indici Caesar Chiper.


Vulnerabilitate


  • Este mai bun din punctul de vedere al securitatii decat Caesar Chiper insa tot slab datorita faptului ca algoritmul se bazeaza pe o simpla substitutie. Cu resursele informationale din ziua de azi un astfel de algoritm devine inutil datorita analizei frecventei literelor. (Deoarece algoritmul se bazeaza pe o simpla substitutie, asta inseamna ca frecventa de aparitie a literei 'a' de exemplu este aceasi cu litera 'k' sa presupunem in alfabet).

  • Brute-force: 1-2 zile

Playfair Chiper

Playfair Chiper face parte din clasa algoritmilor simetrici care a fost si prima tehnica de cripatare care foloseste o diagrama de substitutie.

Algoritmul consta in construirea unei matrici 5x5 in care se primeste ca input plaintextul si un string pentru cheie de encriptare aceasta folosindu-se la construirea matricei. Se extrage pe rand fiecare litera din cheie unica si se introduce in matrice pe pozitiile 1, 2, 3 ... pana la terminarea tuturor literelor unice din cheie apoi completandu-se restul matricei cu restul literelor din alfabet care nu faceau parte din string-ul cheii. Dupa construirea matricei, se ia fiecare litera din plaintext si se cauta in matrice apoi se shifteaza o pozitie la stanga/dreapta (dupa implementare), daca dupa shiftare iese din matrice se ia capatul opus.

Exemplu:


Plaintext: Hello World


Cheie: Secret


SECRT
ABDFG
HI/JLMN
OPQRU
VWXYZ

Plaintext encriptat: ICMP XPUMF

Vulnerabilitate


  • Desi se scapa de dependenta analizei alfabetice tot numarul de matrici este mic fiind doar de dimensiune 5x5.

  • Brute-force: Mai putin de 1 zile

Hill Chiper

Hill Chiper face parte din clasa algoritmilor substituiti, avand baza in algebra liniara. Algoritmul a fost primul care putea folosi in practica trei simboluri deodata in timpul procesului de encriptare si decriptare.

Input-ul pentru acest algoritm consta in plaintext splitat in elemente de cate trei litere, precum si o matrice random inversabila de 3x3 (mod 26) aceasta putand fi construita din valoarea literelor cheiei insa avandu-se grija ca matricea rezultata sa fie inversabila.

Algoritmul va efectua operatia de inmultire dintre matricea 3x3 si fiecare vector de 3 litere apoi rezultatului aplicandu-se modulo 26. Pentru decriptarea procesul e identic doar ca in loc de matricea folosita la encriptare se va folosi inversa acesteaia.

Exemplu:


Plaintext: ACT


Cheie: GYBNQKURP

Plaintext encriptat: FIN

Vulnerabilitate


  • Acasi problema ca la Playfair Chiper, numarul relativ mic de matrici, si in plus nevoia ca matricea sa fie inversabila.

  • Brute-force: Mai putin de 1 zile

Xor Cipher

Operatia pe biti 'xor' este intens folosita in domeniul criptografic alaturi de permutari si substitutii. Aceasta criptare este una relativ simpla. Se cere ca input un plaintext si un string pe post de cheie. Cheia se expandeaza pana la dimensiunea plaintextului. Dupa expandarea cheii se efectueaza xor simplu caracter cu caracter.


Exemplu cod python:

plaintext = input ("Plaintext: ")
cheie = input ("Cheie: ")
plaintext_criptat = [0 for x in range ( len (plaintext) ) ]
plaintext_decriptat = [0 for x in range ( len (plaintext) ) ]
while len (plaintext) >= len (cheie) * 2 :
  cheie = cheie + cheie
i = 0
while len (plaintext) >= len (cheie) :
  cheie = cheie + cheie
  i = i + 1
for index in range ( len (plaintext)) :
  plaintext_criptat[index] = str ( ord (plaintext[index]) ^ ord (cheie[index]))
for index in range ( len (plaintext)) :
  plaintext_decriptat[index] = chr ( int (plaintext_criptat[index]) ^ ord (cheie[index]))

print (plaintext_criptat)
print (plaintext_decriptat)
